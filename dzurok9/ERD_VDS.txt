Разработать полную ERD домена, по UML-диаграмме из урока


Сущности и атрибуты:

User (Пользователь):

UserID (PK): Уникальный идентификатор пользователя.
Phone: Номер телефона пользователя.

Order (Заказ):

OrderID (PK): Уникальный идентификатор заказа.
UserID (FK): Идентификатор пользователя, связь с сущностью User.
VDSID (FK): Идентификатор виртуального сервера, связь с сущностью VDS.
TotalPrice: Итоговая стоимость заказа.
HDD: Объем жесткого диска в гигабайтах.
IPAddressID (FK): Идентификатор IP адреса, связь с сущностью IPAddress.
RAM: Объем оперативной памяти в гигабайтах.
CPUID (FK): Идентификатор процессора, связь с сущностью CPU.
OSID (FK): Идентификатор операционной системы, связь с сущностью OperatingSystem.

VDS (Виртуальный сервер):

VDSID (PK): Уникальный идентификатор виртуального сервера.
CPUID (FK): Идентификатор процессора, связь с сущностью CPU.
RAMID (FK): Идентификатор оперативной памяти, связь с сущностью RAM.
HDDID (FK): Идентификатор жесткого диска, связь с сущностью HDD.
OSID (FK): Идентификатор операционной системы, связь с сущностью OperatingSystem.
Price: Цена за аренду виртуального сервера.

OperatingSystem (Операционная система):

OSID (PK): Уникальный идентификатор операционной системы.
Name: Название операционной системы.

IPAddress (IP адрес):

IPAddressID (PK): Уникальный идентификатор IP адреса.
Address: IP адрес.

CPU (Процессор):

CPUID (PK): Уникальный идентификатор процессора.
Model: Модель процессора.

RAM (Оперативная память):

RAMID (PK): Уникальный идентификатор оперативной памяти.
Size: Объем оперативной памяти в гигабайтах.
HDD (Жесткий диск):

HDDID (PK): Уникальный идентификатор жесткого диска.
Size: Объем жесткого диска в гигабайтах.


User связан с Order по полю UserID (один ко многим), что означает, что каждый пользователь может разместить несколько заказов.

Order связан с User по полю UserID, с VDS по полю VDSID, с IPAddress по полю IPAddressID, с CPU по полю CPUID, с RAM по полю RAMID, и с OperatingSystem по полю OSID.

VDS связан с CPU по полю CPUID, с RAM по полю RAMID, с HDD по полю HDDID, и с OperatingSystem по полю OSID. Каждый VDS имеет только одну операционную систему, одну оперативную память, один жесткий диск, один IP адрес и один процессор.

GET запросы:

Получение информации о пользователе:

URL: /api/users/{UserID}

Этот GET запрос будет получать информацию о пользователе с указанным UserID.

Получение информации о заказе:

URL: /api/orders/{OrderID}

Этот GET запрос будет получать информацию о заказе с указанным OrderID.

Список всех заказов пользователя:

URL: /api/users/{UserID}/orders

Этот GET запрос будет возвращать список всех заказов, принадлежащих конкретному пользователю.

POST запросы:

Создание нового заказа:

URL: /api/orders

Тело запроса (JSON):

json
Copy code
{
    "UserID": 123,
    "VDSID": 456,
    "TotalPrice": 1000.0,
    "HDD": 100,
    "IPAddressID": 789,
    "RAM": 8,
    "CPUID": 101,
    "OSID": 201
}
Этот POST запрос будет создавать новый заказ с указанными характеристиками и возвращать созданный OrderID.

Обновление информации о заказе:

URL: /api/orders/{OrderID}

Тело запроса (JSON):

json
Copy code
{
    "TotalPrice": 150.0
}
Этот POST запрос будет обновлять информацию о заказе с указанным OrderID, например, изменяя итоговую стоимость.

Удаление заказа:

URL: /api/orders/{OrderID}

Этот POST запрос будет удалять заказ с указанным OrderID.
