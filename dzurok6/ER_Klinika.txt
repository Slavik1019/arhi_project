ER-диаграмма для БД клиники, докторов и специализаций:

Сущности и их атрибуты:

Клиника (Clinic):

clinic_id (Primary Key) - Уникальный идентификатор клиники.
Name - Название клиники.
Address - Адрес клиники.
Phone - Телефон клиники.


Пациент (Patient):

patient_id (Primary Key) - Уникальный идентификатор пациента.
First Name - Имя пациента.
Last Name - Фамилия пациента.
Date of Birth - Дата рождения пациента.
Gender - Пол пациента.
Address - Адрес пациента.
Phone - Телефон пациента.


Специализация (Specialization):

specialization_id (Primary Key) - Уникальный идентификатор специализации.
Name - Название специализации.
Description - Описание специализации.


Доктор (Doctor):

doctor_id (Primary Key) - Уникальный идентификатор доктора.
First Name - Имя.
Last Name - Фамилия.
Date of Birth - Дата рождения.
Gender - Пол.
Phone - Телефон.


Время (TimeSlot):

time_slot_id (Primary Key) - Уникальный идентификатор временного слота.
DateTime - Дата и время доступного временного слота.
Duration - Длительность приема.


Запись (Appointment):

appointment_id (Primary Key) - Уникальный идентификатор записи.
patient_id (Foreign Key) - Ссылка на пациента, связанного с данной записью.
doctor_id (Foreign Key) - Ссылка на доктора, связанного с данной записью.
time_slot_id (Foreign Key) - Ссылка на временной слот, связанный с данной записью.
Status - Статус записи (например, "записан", "перенесен", "отменен").
Comment - Комментарий к записи.


Связи между сущностями:

Один ко многим: Клиника -> Доктор (Клиника может иметь множество докторов, но доктор принадлежит одной клинике).
Один ко многим: Доктор -> Специализация (Доктор может иметь множество специализаций, но специализация принадлежит одному доктору).
Один ко многим: Доктор -> Время (Доктор может иметь множество доступных временных слотов для приема, но каждый временной слот принадлежит одному доктору).
Один ко многим: Пациент -> Запись (Пациент может иметь множество записей, но каждая запись принадлежит одному пациенту).
Один ко многим: Доктор -> Запись (Доктор может иметь множество записей, но каждая запись связана с одним доктором).
Один ко многим: Время -> Запись (Время может быть связано с множеством записей, но каждая запись связана с одним временем).

Взаимодействие сущностей:

Пациент и Специализация:

Пациент выбирает специализацию, которая интересует его из списка доступных специализаций.

Пациент и Доктор:

Пациент выбирает доктора из списка доступных докторов в выбранной специализации.
Это может быть сделано на основе имени доктора, его специализации, рейтинга и других факторов.

Пациент и Время:

Пациент выбирает свободное время для записи у выбранного доктора.
Время представляется в виде временных слотов, и пациент может выбирать из доступных слотов.

Пациент и Запись:

Пациент создает запись, связывая ее с выбранным доктором, выбранным временем и своими данными.
Статус записи устанавливается как "записан".
В записи также могут быть указаны дополнительные сведения, такие как цель визита и комментарий.

Пациент и Управление Записью:

Пациент может перенести запись Это изменяет связь записи с временем.
Пациент также может отменить запись, устанавливая соответствующий статус.

Доктор и Специализация:

Доктор может иметь несколько специализаций, что позволяет ему оказывать различные медицинские услуги в разных областях.
Это связь "многие-ко-многим" между докторами и специализациями.

Доктор и Клиника:

Доктор может работать в разных клиниках, что позволяет клиникам иметь множество докторов.
Это также связь "многие-ко-многим".
