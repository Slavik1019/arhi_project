Разработать полную ERD:
Необходимо спроектировать сервис бронирования столика в ресторане. C полями ввода данных о пользователе и заказе, возможностью выбора столика и с кнопкой подтверждения заказа.

User (Пользователь)

UserID (Идентификатор пользователя, уникальный ключ)
Name (Имя пользователя)
PhoneNumber (Контактный номер телефона пользователя)

Reservation (Бронь)

ReservationID (Идентификатор брони, уникальный ключ)
UserID (Связь с таблицей User)
TableID (Связь с таблицей Table)
DateTime (Дата и время бронирования)
NumberOfPersons (Количество персон)

Table (Столик)

TableID (Идентификатор столика, уникальный ключ)
Name (Название столика или его номер)
Status (Свободно, Забронировано, Другое)


Методы в Model для работы с данными:

loadTables(): Этот метод будет выполнять запрос к базе данных для получения списка столиков и их статусов. Он вернет список столиков с их статусами.

bookTable(userID, tableID, dateTime, numberOfPersons): Этот метод будет добавлять запись о бронировании в базу данных. Он примет ID пользователя, ID столика, дату и время бронирования, и количество персон. Затем он добавит новую запись в таблицу бронирований.

changeTable(reservationID, newTableID): Этот метод позволит пользователю изменить бронь на другой столик. Он примет ID бронирования и новый ID столика, затем обновит запись в таблице бронирований с новым столиком.

Далее, в Presenter:

showTables(): Этот метод будет вызывать loadTables() из Model и передавать данные во View для отображения списка столиков.

getUserInfo(): Этот метод будет вызывать View для сбора данных о пользователе (имя, номер телефона, время, количество персон).

confirmReservation(userID, tableID, dateTime, numberOfPersons): Этот метод будет вызываться, когда пользователь подтверждает бронирование. 
Он будет выполнять следующие шаги:

Проверит корректность данных,
если данные некорректны, запросит у View показать сообщение об ошибке,
если данные корректны, запросить у View отобразить прогресс-бар,
вызовет bookTable() из Model для сохранения данных о бронировании.
После завершения бронирования, запросит у View убрать прогресс-бар.
Запросит у Model данные о подтверждении бронирования и передаст их View.
Попросит View отобразить новые данные.
